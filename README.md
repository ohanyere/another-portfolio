# Platform and Backend Engineering Portfolio

Personal engineering website for Ohanyere Francis, repositioned from a frontend portfolio into a platform/backend-focused professional site.

## Positioning

The site emphasizes:

- Backend systems and API-oriented thinking
- Infrastructure automation and deployment workflows
- Docker image building and containerized application direction
- Kubernetes / EKS learning and usage direction
- Terraform infrastructure provisioning concepts
- CI/CD, release validation, and operational readiness
- Observability and reliability fundamentals

Frontend experience is still represented, but as a supporting skill rather than the main professional identity.

## Tech Stack

- React
- TypeScript
- Vite
- Tailwind CSS
- Framer Motion
- Lucide React
- React Router

## Main Sections

- Hero: platform/backend/cloud-native positioning
- Technical Focus: categorized capabilities across infrastructure, containers, backend, DevOps, reliability, and supporting frontend skills
- Platform & DevOps Capabilities: Docker, CI/CD, deployments, Kubernetes/EKS, Terraform, and observability
- Selected Engineering Work: direct project cards with GitHub-first links, concise technical descriptions, stacks, and optional live deployments
- About: backend and infrastructure direction
- Reliability & Delivery: testing philosophy, CI/CD, deployment validation, and reliability mindset
- Contact: role-aligned outreach

## Development

Install dependencies:

```bash
npm install
```

Run locally:

```bash
npm run dev
```

Build:

```bash
npm run build
```

Lint:

```bash
npm run lint
```

## Notes

This project intentionally avoids claiming fake production experience. Some platform capabilities are framed as direction, practice, and project evolution where the underlying project does not yet include full production infrastructure.
